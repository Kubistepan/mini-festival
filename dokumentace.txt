POPIS FESTIVALU
================================================================================

Identita festivalu:
  Název:     Mini festival
  Ročník:    1. ročník
  Lokalita:  Údolní areál u řeky (fiktivní lokalita)
  Termín:    19.–21. června 2026 (3 dny)
  
Charakteristika:
  Komorní, multižánrový třidenní festival hudby a edukativních talků.
  Cíl: mladí dospělí a studenti zajímající se o hudbu, technologie a inovaci.
  
Žánry a obsah:
  - Hudba: indie, folk, elektronika, alt-rock, akustika, soul, jazz, ambient, experimental
  - Formáty: 48 koncertů, 4 workshopy, 7 talků, 1 info položka
  - Jazyky: česky, anglicky
  
Místa:
  - Main Meadow (1200 míst, venkovní)
  - Workshop Barn (250 míst, krytá)
  - Forest Stage – Lesní scéna (500 míst, venkovní)
  
Účinkující: 60 kapel/DJ/artist/speaker/workshop programů
Programových položek: 60 (koncerty, workshopy, talky, info)

1. POSTUP TVORBY
================================================================================

Fáze 1: Specifikace festivalu

  - Definice cílového publika a charakteru akce
  - Plánování webových stránek a API endpointů

Fáze 2: Datová základna
  - Vytvoření festival.xml s minimálními entitami:
    * Festival (identita, vstupenky)
    * 2 místa (venues)
    * 8 účinkujících (performers)
    * 10 programových položek (events)
  - Validation přes festival.xsd s vlastními omezeními

Fáze 3: XML Schéma (XSD)
  - Definice jednoduchých typů s pattern/enum omezeními
  - ID formáty: FEST-***, VEN-***, PERF-***, EVT-***
  - Enumerace pro typy míst a účinkujících

Fáze 4: Web (HTML + CSS)
  - Sémantické HTML5 (header, nav, main, section, article, footer)

  - Jednoduché CSS baseline (bez frameworků)

Fáze 5: API - Transformace
  - 7 XSLT transformací: festival → JSON
  - Funkční Saxon HE 10 transformace (příkazy uloženy v transformations.md)
  - Generování 8 JSON výstupů odpovídajících REST endpointům

Fáze 6: API - Schémata
  - 7 JSON Schema souborů (JSON Schema Draft 2020-12)
  - Validace struktur JSON odpovědí
  - Pattern validation pro ID, format pro datumy, atd.

Fáze 7: API - OpenAPI Specifikace
  - openapi.yaml popis všech 7 endpointů
  - Parametry, odpovědi, HTTP kódy
  - Odkazy na JSON schémata ($ref)

4. KRITICKÉ POSOUZENÍ VYUŽITÍ AI
================================================================================

Přínosy AI v projektu:
  ✓ Rychlé generování XML struktury se správnou well-formed syntaxí
  ✓ XSD omezení (pattern, enum) byla konzistentní a správná
  ✓ HTML sémantika i metadata (Open Graph, JSON-LD) byla kvalitní
  ✓ XSLT transformace fungovaly a správně generovaly JSON
  ✓ JSON schémata obsahovala relevantní validace (pattern, format, required)
  ✓ OpenAPI specifikace byla úplná a validní

Nedostatky a limity AI:
  ~ XML schéma (XSD) mohla být komplexnější (more complex types)
  ~ Některé XSLT transformace nemusily být zcela optimální pro velké datasety
  ~ CSS baseline byl velmi minimalistický (ale zadání to vyžadovalo)
  ~ Některé OpenAPI schémata by mohla mít lepší popis (description field)
  ~ Nebylo prověřeno, zda se transformace chování správně na všech edge-cases

Own contribution (ruční úpravy a ověření):
  - Ověření well-formedness XML a XSD
  - Testování XSLT transformací přes Saxon HE
  - Manuální kontrola HTML validity (sémantika, hierarchie)
  - Manuální validace JSON schémat a jejich shody s výstupy
  - Dokumentace a organizace projektu
  - Kontrola, že všechny soubory existují a jsou ve správných lokacích

Závěr na AI:
  AI byl velmi užitečný pro vytvoření inicální struktury a obsahu, ale ruční
  ověření a validace byla kritická pro kvalitu. Projekt kombinuje AI generaci
  s lidským porozuměním požadavků a testováním.

ZÁVĚR
================================================================================

Projekt Mini-Festival splňuje všechny hlavní požadavky zadání:

1. Festival je fiktivní, vymyšlený, s konzistentní identitou (3 dny, 60+ účinkujících)
2. Datová základna je kompletní a validní (XML, XSD) – 60 eventů, 60 performerů, 3 místa
3. Web je sémantický, validní a dobře strukturovaný – 4 stránky s úplným programem
4. API je popsáno OpenAPI specifikací s funkčními transformacemi
5. Dokumentace je rozsáhlá a pokrývá metodiku a AI posouzení

Rozšíření dat (leden 2026):
  - Původně: 2 dny, 8 účinkujících, 10 eventů
  - Nyní: 3 dny, 60 účinkujících, 60 eventů, 3 místa
  - Změněné soubory: festival.xml, program.html, performers.html, specification.md

Projekt demonstruje schopnost:
  - Pracovat s XML/XSD/XSLT
  - Psát sémantické HTML5
  - Navrhovat a dokumentovat REST API
  - Používat AI jako pomocný nástroj s kritickým zhodnocením

Doporučení pro budoucí vylepšení:
  - Rozšíření web designu (CSS Grid, Flexbox)
  - Přidání dynamického JavaScriptu pro filtrování
  - Implementace skutečného API serveru (Node.js, Python, apod.)
  - Zlepšení XSD s komplexnějšími typy a omezeními
  - Testování XSLT na větších objemech dat
